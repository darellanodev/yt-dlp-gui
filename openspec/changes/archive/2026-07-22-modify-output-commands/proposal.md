## Why

The yt-dlp commands generated by the GUI are outdated. The user now relies on newer yt-dlp features — `--js-runtimes deno`, `--merge-output-format mp4`, and format selection strings with height/codec filtering (`bv*[vcodec^=avc][height<=X]+ba[acodec=aac]`) — that the app doesn't produce. The current output is a simple `best[ext=mp4]/best[ext=aac]` pattern that no longer matches real-world usage.

## What Changes

- **Format selection** updated from `best[ext=mp4]/best[ext=aac]` to `bv*[vcodec^=avc][height<=X]+ba[acodec=aac]/b[ext=mp4][height<=X]/b[height<=X]` for video
- **New flag**: `--js-runtimes deno` added to all video commands
- **New flag**: `--merge-output-format mp4` added to all video commands
- **Cookies optional**: `--cookies-from-browser firefox` becomes a checkbox (default off) instead of always-on
- **Quality becomes resolution-based**: Normal/High radio buttons replaced with 1080p / 720p / 480p
- **Output folder removed**: The folder name input is deleted from the UI and command generation
- **Playlist output template** updated to `%%(playlist_title)s/%%(playlist_index)03d - %%(title)s.%%(ext)s` (video) and `%%(playlist_title)s/%%(playlist_index)s - %%(title)s.%%(ext)s` (audio)
- **Binary name** changed from `yt-dlp.exe` to `yt-dlp`
- **Dead code cleaned up**: `ParamBuilder.processPlaylist()` is either wired in or removed

## Capabilities

### New Capabilities
- `command-generation`: How yt-dlp command strings are assembled — format selection, flags, output templates, and the conditional logic that ties them together

### Modified Capabilities
- `zustand-state`: Store shape changes — quality values change from `'normal'|'high'` to `'1080'|'720'|'480'`, new `cookies: boolean` field added, `folderName` removed

## Impact

- **Core files**: `src/utils/ParamBuilder.ts`, `src/utils/CommandBuilder.ts`, `src/store/useAppStore.ts`, `src/App.tsx`
- **Test files**: `tests/ParamBuilder.test.ts`, `tests/CommandBuilder.test.ts` — must be updated for new format strings, new flags, and removed fields
- **No new dependencies**: All changes are within existing TypeScript/React code
- **No API changes**: The app generates display-only command strings; no external interfaces affected
